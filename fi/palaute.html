Palaute ja vikailmoitukset

Kerro virheestä tai ehdota parannusta. Lähetys avaa GitHubin, jossa ilmoitus tallentuu tähän repoon.

Lokikirja selaimessa

Katselimen työkalupalkin Lokikirja-painike lähettää samanlaiset ilmoitukset automaattisesti GitHub-issueihin — ilman että käyttäjä kirjautuu GitHubiin. Ylläpitäjä konfiguroi teknisen yhteyden (token tai worker); ohje kehittäjille.

Miten tämä toimii

  1. Täytä alla oleva lomake.
  2. Painike avaa GitHubin esitäytettynä issue-luontisivuna.
  3. Kirjaudu GitHubiin (ilmainen tili riittää) ja vahvista lähetys.
  4. Ilmoitus näkyy repossa Issues-listassa.

Sivustolla ei ole evästeitä eikä seurantaa. Lomake ei lähetä mitään ennen kuin vahvistat issuen GitHubissa.

Tietosuoja (GDPR)

Palautelomake ei kerää tietoja itse. Kun vahvistat issuen GitHubissa, tallennamme vain sen, mitä itse kirjoitat ja mitä GitHub liittää tiliisi (käyttäjänimi, aikaleima). Issue on julkinen, ellei projektia myöhemmin suljeta.

  • Älä kirjoita nimeä, osoitetta, henkilötunnusta tai terveystietoja.
  • GitHub käsittelee tietoja Yhdysvalloissa — katso GitHubin tietosuojaseloste.
  • Voit pyytää issuen poistamista ottamalla yhteyttä ylläpitäjään tai sulkemalla issuen itse.

Vaihtoehto ilman GitHub-tiliä: lähetä sähköpostia ylläpitäjälle (katso GitHub-profiili) — älä lähetä arkaluonteisia tietoja sähköpostissakaan.
